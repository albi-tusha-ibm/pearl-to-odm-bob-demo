PROMPT: Create Parity Validation Tools and Comprehensive Documentation

Build utilities to validate parity between legacy and ODM systems, and create comprehensive documentation for the modernization project.

REQUIREMENTS:

## Parity Validation Tool (parity_check.py)
Create a Python script that:
- Reads all 60 test loan applications from legacy_perl/samples/
- Reads expected decisions from expected_decisions.csv
- Simulates legacy PERL-DSL rule execution
- Simulates ODM rule execution (using the design specifications)
- Compares decisions, rate adjustments, documentation requirements, and exceptions
- Generates detailed parity report with:
  * Overall match rate (target: 98.3% = 59/60 matches)
  * Match rates by decision category (Approve, Refer, Decline)
  * Match rates by rule type (Eligibility, Pricing, Documentation, Exceptions)
  * Detailed analysis of mismatches with root cause
  * Summary statistics and recommendations

Technical requirements:
- Use Python standard library only (no external dependencies)
- Clear, well-commented code
- Modular design with separate functions for each rule category
- JSON input/output for easy integration
- Command-line interface with options

## ODM API Demonstration Tool (invoke_odm_stub.py)
Create a Python script that:
- Demonstrates how to invoke ODM Decision Service REST API
- Provides stub implementation (since actual ODM is not deployed)
- Shows request/response format
- Includes example payloads for all decision types
- Documents API endpoints, authentication, headers
- Provides error handling examples

Technical requirements:
- Use Python standard library only (urllib, json)
- Clear documentation of API contract
- Example requests for Approve, Refer, Decline scenarios
- Response parsing and display
- Extensible for actual ODM integration
